# 編輯分店表單：客戶編碼唯讀欄 — 設計

日期：2026-05-19
狀態：已核准，待實作

## 1. 目的

編輯分店資訊時，目前看不到該分店的客戶編碼。編碼影響會計 AR/AP，操作者
需要在編輯時看得到當下編碼，避免誤判（例如不確定該分店有沒有發過碼、
或改路線前想先確認舊碼）。

## 2. 需求

在「編輯分店資訊」表單，於「路線（配送區域）」欄位所在那一列下方，
新增一個唯讀欄位顯示目前客戶編碼。

## 3. 設計

**位置**：`CustomersPage.tsx` 編輯分店表單 JSX，「電話 / 路線」2 欄
grid 那一列（約 line 645-657）下方，新增一整列 full-width。

**欄位內容**：
- 標籤「客戶編碼」。
- 唯讀顯示框：
  - 有編碼 → 顯示編碼字串（如 `C01`），mono 藍字。
  - 無編碼 → 顯示「（未發碼）」灰字。
- 欄位下方小字：「系統自動管理，改路線時自動發碼/封存，不可手動編輯」。

**資料來源**：直接讀 `editTarget.record.ref`。

**行為**：純唯讀。
- 不納入 `editBranch` state。
- 不進 `saveEdit` 的儲存 payload。
- 表單開啟時顯示當下編碼；若本次編輯改了路線並儲存，需重開表單才會
  看到新碼（編碼由 `assign_customer_code` / `reassign_customer_route`
  action 寫入，非此欄位職責）。

## 4. 不做（YAGNI）

- 不顯示 `code_history` 歷史與封存日期。
- 不可編輯。
- 不加複製按鈕。

## 5. 影響範圍

- 僅改 `vfs/admin/src/pages/admin/CustomersPage.tsx` 編輯分店表單一段 JSX
  （約 8 行）。
- 無新 action、無資料模型變更、無 `db_admin.py` 變更。

## 6. 驗證

- 前端改動，需 publish 後於瀏覽器實測：
  - 編輯一家有編碼的分店 → 路線欄下方顯示該編碼。
  - 編輯一家未發碼的分店 → 顯示「（未發碼）」。
